# Customer Dashboard Workflow

## Purpose

Provide a customer-safe operational dashboard with clear next actions and no admin/internal controls.

## Dashboard Scope

- Active contracts and plan posture
- Upcoming dues and payment history
- Delivery visibility
- Support/service request entry points
- Lucky draw status visibility from customer-safe data
- Notification summary and recent customer alerts

## Quick Actions

- View Contracts
- View Due / Payments
- Download Receipts
- Track Delivery
- Raise Support Request
- Update Profile

## Guardrails

- No admin finance/reversal/reconciliation controls
- No vendor or partner internals
- No accounting posting internals
